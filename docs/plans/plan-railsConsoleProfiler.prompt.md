### Plan: Rails Console Performance Profiler

A Ruby gem that profiles block execution, measuring wall time, database query counts, and memory usage, displaying the results in a formatted table. Works in plain Ruby contexts; Rails integration (SQL tracking, console helpers) is loaded automatically when Rails is present.

**Steps**
1. **Gem Initialization & Setup**
   - Initialize a new Ruby gem (e.g., `bundle gem rails_console_profiler`).
   - Add hard dependencies to the `.gemspec`: `terminal-table` and `memory_profiler`.
   - Do **not** add `railties` or `activerecord` as hard dependencies — Rails is optional.
2. **Implement Core Profiler Logic**
   - Define a top-level `ConsoleBenchmark` module (standalone, does not conflict with Ruby's standard `Benchmark` library).
   - Implement `ConsoleBenchmark.measure(&block)` as a module method.
   - Inside `measure`:
     - Track time using `Process.clock_gettime(Process::CLOCK_MONOTONIC)`.
     - Track SQL queries dynamically via `ActiveSupport::Notifications.subscribed(callback, "sql.active_record")` — **only when `ActiveSupport::Notifications` is defined**. Keep a simple counter (ignore `SCHEMA` or `CACHE` queries if desired). When Rails is absent, SQL count is omitted from results.
     - Track memory via `MemoryProfiler.report(&block)`.
   - Combine these metrics into a structured result object or Hash.
3. **Implement Table Output Formatter**
   - Create a `RailsConsoleBenchmark::Formatter` class.
   - Use `Terminal::Table` to accept the structured result object and format it into a clear grid (e.g., columns for: Wall Time (ms), SQL Queries, Total Allocated Memory, Total Retained Memory).
4. **Optionally Integrate with Rails Console**
   - In the main `require` file, conditionally load the Railtie: `require_relative "rails_console_benchmark/railtie" if defined?(Rails::Railtie)`.
   - The Railtie (`RailsConsoleBenchmark::Railtie`) extends `Rails::ConsoleMethods` to expose `ConsoleBenchmark` at the top-level console scope — since `ConsoleBenchmark` is already a top-level constant once the gem is required, no extra aliasing is needed.
   - When Rails is absent, `ConsoleBenchmark.measure { ... }` works identically in any Ruby context.

**Relevant files**
- `rails_console_profiler.gemspec` — Declare `terminal-table` and `memory_profiler` as hard dependencies. Rails gems are **not** listed as dependencies.
- `lib/rails_console_profiler/tracker.rb` — Implements `ConsoleBenchmark.measure` as a module method, combining `MemoryProfiler` and monotonic time tracking. SQL tracking via `ActiveSupport::Notifications` is enabled only when `defined?(ActiveSupport::Notifications)`.
- `lib/rails_console_profiler/formatter.rb` — Transforms raw outputs into a `Terminal::Table`.
- `lib/rails_console_profiler/railtie.rb` — Hooks into `Rails::Railtie` to mix the `measure` method into `Rails::ConsoleMethods`. Only required when `Rails::Railtie` is defined.

**Verification**
1. **Without Rails**: Run `bin/console` and execute `ConsoleBenchmark.measure { 1 + 1 }`. Verify the table shows time and memory metrics (SQL column absent or shown as N/A).
2. **With Rails**: Embed within a dummy Rails app's `rails c`. Execute `ConsoleBenchmark.measure { User.last }` and verify the table appears with non-zero metrics for all three categories (Time, DB Queries, Memory Allocation).
3. Validate that `SCHEMA` DB queries generated by Rails during queries aren't inaccurately skewing the query count by excluding them in the subscriber logic.

**Decisions**
- Public API is `ConsoleBenchmark.measure { ... }` — a top-level `ConsoleBenchmark` module defined by this gem.
- Implemented as a block-based runner (`ConsoleBenchmark.measure { ... }`) based on your preference.
- Used `terminal-table` for highly readable inline console metrics.
- Leveraged `memory_profiler` for out-of-the-box advanced object allocation tracking.
- Rails is **optional**: the Railtie and SQL tracking are only activated when `Rails::Railtie` / `ActiveSupport::Notifications` are defined, making the gem safe to use in non-Rails Ruby projects.
- Rails console integration (via `Rails::ConsoleMethods`) is provided automatically when Rails is present for an improved developer experience.
